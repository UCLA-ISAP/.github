# UCLA ISAP Branding Guide

A quick reference for anyone contributing to repositories under this organization.

## Name

The full name is **UCLA Integrated Substance Use and Addiction Programs (ISAP)**. After the first mention in a document, "UCLA ISAP" or "ISAP" is fine.

Note: The program was previously called "Integrated Substance Abuse Programs." Some older materials and publications still use that name. New work should use the current name.

## Language

Use person-first language. Write "people with substance use disorders," not "addicts" or "substance abusers." Prefer "substance use" over "substance abuse" in line with current clinical standards and APA style.

Keep writing clear and direct. We're an academic research group, not a marketing department — avoid buzzwords and vague claims. If something is effective, cite the evidence.

## Colors

Follow the [UCLA Brand Guidelines](https://brand.ucla.edu/). The primary palette:

| Color      | Hex       | Usage                          |
|------------|-----------|--------------------------------|
| UCLA Blue  | `#2774AE` | Primary brand color, headers   |
| UCLA Gold  | `#FFD100` | Accent, highlights             |
| White      | `#FFFFFF` | Backgrounds                    |
| Dark Gray  | `#333333` | Body text                      |

Do not use off-brand blues or substitute colors. When in doubt, check [brand.ucla.edu](https://brand.ucla.edu/).

## Logos and Visual Assets

We do not host logo files in this repository. For UCLA identity marks, refer to the [UCLA Brand Guidelines](https://brand.ucla.edu/identity/logos-marks). Any ISAP-specific assets should be coordinated through the program's communications staff.

## Repository Conventions

- Each repo should have a README that explains what the project is and how to use it.
- Include a license file where appropriate. For publicly funded research code, consider open-source licenses (MIT, Apache 2.0, etc.) — check with your PI.
- Reference ISAP by its full name at least once in each repo's README.
- Use clear, descriptive commit messages.
